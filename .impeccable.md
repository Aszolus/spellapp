# SpellApp — Impeccable Design Context

## Design Context

### Users
Pathfinder 2e tabletop players — mix of casual and hardcore, leaning hardcore/min-maxer. Used **at the table during play**, in-session, while the game is running. Often dim room lighting, phone/tablet in one hand, dice/sheet in the other. Speed matters: the user is looking something up *while a turn is waiting on them*.

Primary jobs-to-be-done:
- Quickly find a specific spell by name, trait, or level during combat.
- Verify action cost, range, save, and effect without digging.
- Manage a character's known/prepared list between or during sessions.
- Trust the data is correct (Foundry-sourced, offline, no stale cache surprises).

### Brand Personality
**Fantasy. Precise. Tactical.**

- *Fantasy* — this is a spellbook, not a CRUD app. It should feel like the thing a wizard actually carries.
- *Precise* — rules-exact. No rounding off, no marketing fluff. Numbers and traits are sacred.
- *Tactical* — optimized for lookup under time pressure. Dense is fine; slow is not.

Emotional goals: confidence (the info is right), calm focus (not chaotic or flashy), a little bit of wonder (this is a fantasy game after all).

### Aesthetic Direction
**Arcane / mystical** — old grimoire energy, but tuned for at-table legibility, not decorative cosplay. Think *a well-worn reference tome a careful spellcaster actually uses*, not a Hollywood prop book.

- Paired typography: a distinctive display/serif for spell names and section headers, a calm humanist body face for rules text. No monospace-as-shorthand-for-technical. No banned reflex fonts (Fraunces, Cormorant, Playfair, IBM Plex, Inter, DM Sans family, etc.).
- Palette: muted, slightly warm neutrals tinted toward parchment / candlelight in light mode; ink-on-vellum in dark mode. Tradition accent (traits: arcane, divine, occult, primal) carries color; the UI chrome stays quiet. OKLCH-equivalent perceptual thinking, even though Material 3 ColorScheme is the delivery mechanism.
- Ornament is earned, not sprinkled. A drop-cap on a spell name, a rule-line above a heading, a thin bracket marking action cost — yes. Glassmorphism, gradient text, glowing neon accents, cyan-on-dark, purple-to-blue gradients — no.
- Both light and dark themes required. Dark is default (dim at-table lighting). User-selectable theming is a known future requirement — keep color roles semantic so alternate themes can swap cleanly.

**Anti-references** — do not clone or echo:
- **D&D Beyond** — card-heavy marketing feel, big hero art, branded accents.
- **Foundry VTT** — dense developer/console aesthetic, tabbed admin panels.
- **Archives of Nethys** — wiki-style HTML tables, utilitarian but visually flat.

The app should be instantly distinguishable from all three on first glance.

### Design Principles
1. **Readable in dim light, one-handed, under pressure.** Touch targets ≥ 48dp. Type hierarchy strong enough to scan without reading. Never hide critical info behind a tap.
2. **Rules-exact over decorative.** If a choice trades legibility of a spell's action cost / range / save for visual flourish, legibility wins. Ornament supports the content; it never competes with it.
3. **Grimoire, not storefront.** Lean into book/manuscript metaphors (rule-lines, drop-caps, margin notes, considered whitespace) rather than SaaS dashboard metaphors (cards-in-grids, metric tiles, icon-above-heading templates).
4. **Semantic theme tokens.** All color/typography flows through Material 3 roles so the planned user-selectable themes can swap without touching screens.
5. **Quiet chrome, loud content.** The app's framing (nav bar, app bar, surfaces) stays restrained. The spell itself — its name, its traits, its effect — is where visual weight lives.

### Technical Constraints
- Android Jetpack Compose, Material 3, min SDK 28, target 35.
- Strictly offline — no network, no remote fonts at runtime (bundle any custom font as a resource).
- No current explicit accessibility requirement beyond Material defaults, but touch targets and contrast remain non-negotiable because of the at-table context.
